HTML

HTML (Hypertext Markup Language) is the main language used to make web pages. It uses tags to organize the content on a page. Here is a simple example of HTML code: 


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Web Page</title>
</head>
<body>
    <h1>Hello World</h1>
    <p>This is my first webpage!</p>
    <p>HTML allows us to structure content effectively.</p>
</body>
</html>

Key HTML Concepts

  • HTML tags define parts of a webpage, like <h1> for titles and <p> for paragraphs.
  • The file starts with <!DOCTYPE html> to show which version of HTML is used.
  • Common parts include <a> for links, <img> for images, and <div> for sections of the page.

CSS

CSS (Cascading Style Sheets) is used to make HTML content look better. CSS helps you set styles like fonts, colors, and layouts to make your webpage look nice. Here is a simple example of a CSS file: 


body {
    background-color: #f0f0f0;
    font-family: Arial, sans-serif;
}

h1 {
    color: #333;
    text-align: center;
}

p {
    color: #666;
    font-size: 16px;
}

Key CSS Concepts

  • CSS selectors (like body, h1) choose which parts of the HTML to style.
  • Each rule has a property (like color) and a value (like "red").
  • CSS can be in its own file (called external), inside the HTML file (called internal), or in the HTML tags (called inline).

GitHub

GitHub is a website where you can save your code and work with others on projects. It uses Git, which is a tool to track changes to code. Here are some common GitHub commands for working with code: 


# Start a new Git project
git init

# Add files to save
git add .

# Save changes with a message
git commit -m "first changes"

# Connect to a GitHub repository
git remote add origin https://github.com/username/repo-name.git

# Upload your changes to GitHub
git push origin main

Git Guide

Git is a tool that helps track changes to files. Here are some common Git commands used when writing software.

1. Initialize a Git Repository


git init

This command creates a new Git repository in your project folder.

2. Add Files to Staging

After modifying files, add them to the staging area


git add

Or add all files


git add .

Then commit your changes with a message


    git commit -m "Your commit message"

3. Branching

Create a new branch to work on a feature separately:


git branch new-feature
git checkout new-feature

4. Merging Branches

Once a feature is completed, merge it into the main branch:


git checkout main
git merge new-feature

5. Pushing to GitHub

To push your code to GitHub, add the remote repository:


git remote add origin https://github.com/username/repo-name.git

Then push to the main branch:


git push origin main

6. Cloning a Repository

To clone a repository from GitHub to your local machine:


git clone https://github.com/username/repo-name.git

How to Use

This project covers the basics of HTML, CSS, Git, and GitHub to help you start with building websites and managing code. Follow the steps and commands provided to create, manage, and work on your projects with others.
